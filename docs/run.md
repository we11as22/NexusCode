# NexusCode — запуск и режимы работы

Один актуальный источник CLI и расширения — **этот репозиторий**. Команда `nexus` из репо поддерживает оба режима: **без сервера** (по умолчанию) и **с сервером** (общая БД, переключение сессий).

---

## 1. Удалить старую версию и поставить актуальную

Если раньше ставили `nexus` глобально не из этого репо (например, другой пакет или старая сборка), лучше удалить и использовать только версию из репо:

```bash
# Удалить старую глобальную установку (если была)
npm uninstall -g nexus 2>/dev/null || true

# Из корня репо NexusCode — одна команда: сборка + глобальная команда nexus + .vsix
cd /path/to/NexusCode
nvm use 20   # обязателен Node 20
pnpm run ready
```

После этого команда `nexus` везде указывает на сборку из репо и поддерживает оба режима (без сервера и с `--server`). Расширение для VS Code: `packages/vscode/nexuscode-0.1.0.vsix` (установить через Extensions → Install from VSIX...).

**Альтернатива по шагам:** `pnpm run setup` → `cd packages/cli && npm link && cd ../..` — тогда `nexus` тоже будет из репо.

---

## 2. Режим без сервера (по умолчанию)

Работает сразу после `pnpm run ready` (или `setup` + `npm link`). Сервер не нужен.

- **CLI:** `nexus` или `nexus "запрос"` — агент в процессе, сессии по `cwd` в `~/.nexus/sessions/`.
- **Расширение:** открой панель NexusCode (Ctrl+Shift+N); в настройках **Server URL** оставь пустым — сессии и сообщения хранятся локально.

---

## 3. Режим с сервером (общая БД, переключение сессий)

Один и тот же `nexus` и расширение могут работать с сервером: общая БД, список сессий, продолжение с `--continue` или `--session <id>`.

### Запуск сервера

Из корня репо (Node 20):

```bash
pnpm serve
```

Сервер слушает **http://127.0.0.1:4097**. БД: `~/.nexus/nexus-server.db` (или `NEXUS_DB_PATH`). Если порт занят: `NEXUS_SERVER_PORT=4098 pnpm run serve` или освободи порт (`lsof -i :4097` → `kill <pid>`).

### Расширение с сервером

1. В панели NexusCode открой **Settings** (шестерёнка).
2. В блоке **NexusCode Server** укажи **Server URL**: `http://127.0.0.1:4097`.
3. Вкладка **Sessions** подгружает список с сервера; сообщения постранично (последние 100).

### CLI с сервером

```bash
nexus --server http://127.0.0.1:4097
# или
export NEXUS_SERVER_URL=http://127.0.0.1:4097
nexus --continue   # последняя сессия с сервера
nexus --session <id>   # конкретная сессия
```

Расширение и CLI при указанном сервере используют одну БД и одни сессии.

---

## 4. Разработка расширения (F5)

1. Открой в VS Code папку **`NexusCode/packages/vscode`**.
2. **F5** (Run → Start Debugging) — окно Extension Development Host с расширением.
3. После изменений: из корня `pnpm build`, в хосте — **Ctrl+R** (Reload Window).

---

## 5. Полезные команды

| Действие | Команда |
|----------|--------|
| Всё в одну команду (сборка + nexus + .vsix) | `pnpm run ready` |
| Сборка всего | `pnpm build` |
| Запуск сервера | `pnpm serve` |
| Упаковка .vsix | `pnpm package:vscode` (Node 20+) |
| CLI без сервера | `nexus` |
| CLI с сервером | `nexus --server http://127.0.0.1:4097` |

Подробнее: **README.md** в корне (требования Node 20, store-dir, NexusCode Server).
