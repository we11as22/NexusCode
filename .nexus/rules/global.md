# NexusCode Global Rules

These rules apply to all NexusCode sessions in this workspace.

## Code Style
- Follow the existing code style and conventions in the project
- Prefer TypeScript strict mode; avoid `any` types when possible
- Use descriptive variable names — avoid abbreviations
- Write clean, readable code. Correctness over cleverness.

## Before Making Changes
- Read and understand the existing code before modifying it
- Check for related tests and update them if needed
- Consider edge cases and error handling

## After Making Changes
- Verify that changes compile without errors
- Check that related tests still pass if test infrastructure is available
- Don't leave TODO comments for trivial items — either fix them or document why

## Communication
- Be concise and precise
- Explain non-obvious decisions
- When something is ambiguous, ask rather than assume
